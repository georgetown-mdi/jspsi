How it works. Each of you keeps your file on your own machine. psilink compares cryptographic fingerprints of the fields you match on - a private set intersection - so only the records you both hold are revealed, and only to the people the terms name. Your browser connects directly to your partner's.

Design notes - Landing

  1. GOV.UK: Start page minimalism. One outcome sentence and two doors. Every configuration choice lives inside the bench, behind safe defaults - the start page carries none of it.
  2. 18F Content Guide: interface writing. Outcome first, mechanism second: "find the records you both hold" precedes any mention of PSI, which appears once, in the explainer strip.
  3. GDS principle 1: Start with user needs. The scheduled-exchange line is a quiet signpost for the operator persona, not a third door - SFTP is a transport chosen later, inside the same setup.

Design notes - States & errors

  1. GOV.UK: Error messages. Specific and blame-free: each message states what happened and the one concrete way forward. No vague failure phrasing, no apology, no jargon.
  2. Never clear input. A failure keeps everything the user typed or chose - the pasted terms, the chosen file, the entered name - so recovery is a correction, not a restart.
  3. Focus lands on the failure. Alerts take tabIndex="-1" and receive focus when they appear; the security failure is worded to stop retries ("Do not retry") while the retryable one invites them - danger and inconvenience never share a voice.
